## 🔄 工作流

### 1️⃣ 收集 (Inbox)
**目标**: 快速捕获，不中断思考
- 任何想法、笔记、链接先丢进 `01-inbox/`
- 使用每日笔记快速记录
- 不担心分类，晚上统一处理

### 2️⃣ 处理 → 概念卡片 (Concepts)
**目标**: 提取核心概念，建立原子笔记
- 从 Inbox 提取有价值的概念
- 创建概念卡片到 `02-concepts/`
- 使用 [[双向链接]] 关联已有概念
- 每个卡片一个核心想法

### 3️⃣ 连接 → MOC (Maps)
**目标**: 建立知识结构，发现关联
- 为每个主题创建 MOC (Map of Content)
- 在 MOC 中索引相关概念卡片
- 使用图谱视图发现隐藏连接
- 定期回顾和更新 MOC

### 4️⃣ 归档 (Archive)
**目标**: 释放空间，保留价值
- 完成的项目 → `05-archive/`
- 过时但仍可能有用的 → 打标签 #archive
- 完全无用的 → 删除

## 🤖 OpenClaw 集成

### 自动处理流程
```
用户发送碎片化信息
    ↓
Claw Mate 分析 → 提取核心概念
    ↓
自动创建/更新概念卡片
    ↓
建立知识连接
    ↓
更新相关 MOC
    ↓
汇报新发现和连接
```

### 可用指令
- "整理到概念库: [内容]" → 创建概念卡片
- "建立连接: [[概念A]] 和 [[概念B]]" → 双向链接
- "查看 [[人工智能]] 的知识网络" → 展示关联
- "今天学到了什么?" → 回顾今日概念

## 📊 图谱视图使用

### Obsidian 图谱设置
- **显示标签**: `#concept`, `#moc`
- **颜色分组**: 按文件夹着色
- **节点大小**: 按链接数量
- **过滤**: 隐藏孤立节点

### 图谱中的发现
- 🔴 大节点 = 核心概念（被引用多）
- 🔵 聚类 = 知识领域
- 🟡 桥梁 = 跨领域连接
- ⚪ 孤儿 = 待建立连接
