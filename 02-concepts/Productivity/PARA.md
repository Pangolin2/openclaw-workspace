# PARA 方法

## 📝 核心定义

> PARA 是 Tiago Forte 提出的知识组织四分类法：Projects（项目）、Areas（领域）、Resources（资源）、Archives（归档）。

## 🔍 详细解释

### 四个分类

#### 📁 Projects（项目）
- **定义**: 有明确目标和截止日期的任务
- **例子**: 完成论文、开发App、准备演讲
- **特点**: 临时性，完成后归档
- **问题**: "现在在进行什么？"

#### 🎯 Areas（领域）
- **定义**: 需要长期维护的责任领域
- **例子**: 健康、职业、学习、财务
- **特点**: 持续性，无明确结束时间
- **问题**: "我需要长期负责什么？"

#### 📚 Resources（资源）
- **定义**: 感兴趣的主题或参考材料
- **例子**: 编程技巧、心理学、旅行攻略
- **特点**: 以备将来参考
- **问题**: "什么可能引起我的兴趣？"

#### 📦 Archives（归档）
- **定义**: 已完成或不活跃的项目/领域
- **例子**: 过去的项目、不再维护的领域
- **特点**: 保留但隐藏，需要时可检索
- **问题**: "什么已完成但仍需保留？"

### 工作流
```
新信息 → 判断类型 → 放入对应文件夹
              ↓
    项目(临时) 领域(长期) 资源(参考)
              ↓
         完成/过时 → Archives
```

## 💡 我的理解

### 关键洞察
- 按"可操作性"分类，而非主题
- Projects 驱动行动，Areas 维护标准
- 避免过度分类的陷阱

### 应用场景
- Obsidian/Notion 文件夹结构
- 邮件管理
- 任务管理

### 常见误区
- ❌ 按主题分类（会混淆项目和资源）
- ❌ 归档太频繁（失去上下文）
- ❌ 领域定义不清（与项目混淆）
